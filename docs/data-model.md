# データ前提と関連

このページでは、業務フローの前提になる主要データを整理します。  
システム上の処理は、主に「証憑」「伝票」「承認」「入出金」「マスタ」の組み合わせで進みます。

## 主要データの関係

```text
ファイルアップロード
  -> OCRキャッシュ
  -> 伝票
       -> 明細
       -> 承認履歴
       -> 取引先
       -> 書類種別
       -> 部門
       -> 支払方法
       -> 会計処理方法
       -> 入出金取引
```

## 1. ファイルアップロード

証憑ファイルそのものを表すデータです。

- ファイル名
- アップロード者
- 発行区分
- OCR 実行対象

1つのアップロードから、通常は1件の伝票が作成されます。

## 2. OCRキャッシュ

アップロードしたファイルに対する OCR 結果の保存領域です。

- ページ番号ごとの認識結果
- 画面で選択した文字範囲
- 推定した取引先候補

基本情報入力時の補助データとして使われます。

## 3. 伝票

業務の中心になるデータです。  
申請から承認、経理処理、支払管理まで、ほぼすべてが伝票単位で進みます。

主な項目:

- 状態
- 発行区分
- 書類種別
- 部門
- 発行日
- 取引先
- 合計金額
- 税区分別金額
- 伝票番号
- 摘要
- 備考
- 申請者名
- 申請日
- 支払期日
- 支払方法
- 会計処理方法
- 明細テンプレート
- 経理承認日
- 支払済日時
- 会計出力日時

## 4. 明細

伝票の会計明細です。  
合計金額と整合するように設定します。

主な項目:

- 決済種別
- 補助コード1〜3
- 金額
- 税額
- 摘要
- 支払依頼内容
- コメント
- 税率
- 税区分

## 5. 承認

承認者がどの伝票をいつ承認したか、却下したかを保持します。

- 承認種別
- 承認者
- 対象伝票または対象取引
- 主承認かどうか
- コメント
- 却下フラグ

## 6. 入出金取引

銀行 CSV から取り込んだ入出金データです。

- 入出金番号
- 取引日
- 摘要
- 金額
- 銀行名
- 支店名
- 取込口座
- 消込済みかどうか

## 7. 会計出力

会計連携用 CSV の出力履歴です。

- 出力ファイル名
- 対象伝票
- 出力者
- 出力日時

## 伝票の状態

コード上では、伝票は次の状態で管理されています。

- 下書き: 入力途中
- 申請中: 承認待ち
- 却下: 修正が必要
- 承認済み: 支払承認完了
- 支払済み: 入出金処理まで完了
- 廃止扱い: 修正履歴の旧版など

## マスタとの関係

伝票入力や承認フローでは、次のマスタが特に重要です。

- 取引先
- 書類種別
- 部門
- 承認者
- 承認上限
- 決済種別
- 会計処理方法
- 支払方法
- 支払期日ルール
- システム設定

マスタが不足していると、申請情報の入力や明細作成で詰まりやすくなります。
