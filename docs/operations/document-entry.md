# 伝票作成と申請

## 1. 基本情報入力

基本情報入力では、証憑から読み取った内容を伝票として整理します。

主な入力項目:

- 書類種別
- 部門
- 発行日
- 取引先
- 合計金額
- 税率別金額
- 伝票番号
- 摘要
- 備考
- タグ

## 基本情報入力の流れ

1. OCR の候補を確認する
2. 書類種別と部門を選ぶ
3. 取引先と日付、金額を確認する
4. 必要に応じてタグや備考を追加する
5. 下書き保存または次へ進む

## 2. 申請情報入力

申請情報入力では、支払と会計に必要な情報を登録します。

主な入力項目:

- 申請者名
- 支払期日
- 支払方法
- 会計処理方法
- 税計算方法
- 明細

## 明細入力の考え方

明細では、伝票合計と一致するように金額を設定します。

- 税込処理: 金額に税相当を含める
- 税抜処理: 本体金額と税額を分けて管理する
- 外税別建て: 明細と税の扱いを別に設計する

## 下書き保存と申請提出

- 下書き保存: 入力途中の状態で保存する
- 申請提出: 承認フローへ回す

申請提出時は、状態が「申請中」になります。

## 申請書 PDF

画面から申請書 PDF を作成できる構成です。  
運用に応じて、伝票画像付きや会計情報付きの出力を使い分けます。

## 差戻し後の再対応

却下された伝票は、内容を修正して再申請します。

確認ポイント:

- 却下コメント
- 金額整合
- 取引先情報
- 支払方法
- 会計処理方法
- 明細内容
