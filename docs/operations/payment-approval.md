# 支払承認

## 支払承認の位置づけ

支払承認は、申請された伝票を部門や承認権限に基づいて確認する工程です。

## 主に使う画面

- 申請承認一覧
- 基本情報確認画面
- 申請情報確認画面
- 承認状況一覧

## 一覧画面で見ること

- 申請中の伝票
- 承認済みの伝票
- 却下された伝票
- 自分が承認すべき対象

## 承認手順

1. 一覧から対象伝票を開く
2. 基本情報を確認する
3. 申請情報と明細を確認する
4. 問題なければ承認する
5. 不備があればコメント付きで却下する

## 却下時の考え方

却下する場合は、申請者が修正しやすいように理由を明確に残してください。

例:

- 取引先が誤っている
- 合計金額と明細金額が一致していない
- 支払方法が不適切
- 添付証憑が不足している

## 修正履歴について

承認中の伝票が修正されると、履歴管理のため旧版が履歴扱いになることがあります。  
最新の有効な伝票を確認する前提で運用してください。

## 承認状況一覧

承認状況一覧では、部門や条件で絞り込みながら進捗を確認できます。

向いている使い方:

- 月次の承認滞留確認
- 部門別の処理状況確認
- 特定期間の承認進捗確認
