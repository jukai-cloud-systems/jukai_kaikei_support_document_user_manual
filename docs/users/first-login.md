# ユーザー作成から初回ログインまで

このページでは、管理者がユーザーを作成してから、利用者が初回ログインするまでの流れを説明します。

![ユーザー作成プレースホルダー](assets/user-create-flow.svg)

## 全体の流れ

1. 管理者がユーザーを作成する
2. 利用者へログイン情報を共有する
3. 利用者が通常ログインを行う
4. 必要に応じてプロフィール情報を更新する

## 管理者側: ユーザー作成手順

管理者権限を持つユーザーが、設定メニューのユーザー管理から作成します。

### 入力する主な項目

- ユーザーID
- 表示名
- メールアドレス
- 初期パスワード
- 初期パスワード確認
- 権限設定
- 所属部門

### 作成手順

1. `設定` メニューを開きます。
2. `ユーザー設定` を開きます。
3. `新規作成` を押します。
4. 必須項目を入力します。
5. 必要な権限をチェックします。
6. 所属部門を選択します。
7. 保存します。

## 利用者へ共有する情報

作成後は、次の情報を安全な方法で利用者へ共有してください。

- ドメイン名
- ユーザーID
- 初期パスワード
- ログインURL

## 利用者側: 初回ログイン手順

1. ログイン画面を開きます。
2. ドメイン名、ユーザーID、初期パスワードを入力します。
3. ログインボタンを押します。
4. メールで届いた OTP を入力します。
5. ホーム画面へ遷移したことを確認します。

## 初回ログイン後に推奨する操作

初期パスワード運用をしている場合は、初回ログイン後にプロフィール画面からパスワード変更を行う運用を推奨します。

![プロフィール変更プレースホルダー](assets/user-profile-placeholder.svg)

### プロフィール画面で変更できる項目

- ユーザーID
- メールアドレス
- パスワード

### パスワード変更時の注意

- 現在のパスワード入力が必要です
- 新しいパスワードと確認入力を一致させてください
- メールアドレス変更時は確認メール運用が入る場合があります

## 初回ログインでつまずきやすい点

- ドメイン名の入力漏れ
- ユーザーIDではなくメールアドレスを入力してしまう
- OTP メールの見落とし
- 管理者が誤ったメールアドレスを登録している
