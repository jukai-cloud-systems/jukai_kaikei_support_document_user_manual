# 権限による違い

このシステムでは、見えるメニューや実行できる操作が、主に次の2つで決まります。

- ユーザー権限フラグ
- 承認者マスタの設定有無

![権限マトリクスプレースホルダー](assets/user-permissions.svg)

## 1. ユーザー権限フラグ

ユーザー作成画面で設定できる主な権限は次の3つです。

### システム管理者

- 各種設定メニューにアクセスできます
- ユーザー管理を利用できます
- 部門、取引先、書類種別、承認設定などの管理に関わります

### 会計処理

- 経理関連メニューの利用対象になります
- 経理承認一覧や銀行 CSV 取込などの機能に関係します
- ただし、実際の承認可否は承認者マスタ設定にも依存します

### 会計管理者

- 既に経理承認済みのデータでも、より強い編集権限を持つ想定の権限です
- 経理確定後の修正が必要な特別運用で使う想定です

## 2. 承認者マスタの有無

ユーザーに権限フラグが付いていても、承認者マスタが無ければ承認できない機能があります。

### 支払承認者として設定されている場合

- 自部門の申請承認一覧で確認対象が表示されます
- 承認、却下、確認の操作が可能になります

### 経理承認者として設定されている場合

- 経理承認一覧の対象が表示されます
- 承認上限内の伝票について確認や承認が可能になります

## 3. 主な違いの整理

| 立場 | 主な条件 | できること |
| --- | --- | --- |
| 一般利用者 | ユーザー登録済み | ログイン、証憑登録、申請、プロフィール変更 |
| 支払承認者 | 承認者マスタに支払承認設定あり | 申請内容確認、承認、却下 |
| 経理担当者 | 会計処理権限あり | 経理メニュー参照、対象機能利用 |
| 経理承認者 | 会計処理権限あり かつ 経理承認者設定あり | 経理承認一覧で確認、承認 |
| 会計管理者 | 会計管理者権限あり | 承認済みデータへの強い編集権限を持つ運用 |
| システム管理者 | システム管理者権限あり | 設定メニュー、ユーザー管理、各種マスタ管理 |

## 4. メニューの見え方の傾向

### 一般利用者

- ホーム
- 伝票一覧
- ファイルアップロード
- 自分が関わる申請一覧

### 承認者

- 一般利用者メニュー
- 申請承認一覧
- 承認状況一覧

### 経理担当者

- 経理承認
- 入出金 CSV 取込
- 会計出力

### システム管理者

- 設定メニュー全般
- ユーザー設定
- 各種マスタ設定

## 5. 注意点

- `会計処理` 権限だけでは、必ずしもすべての経理承認ができるわけではありません
- 支払承認や経理承認は、承認上限金額にも影響を受けます
- 同じユーザーが、一般利用者と承認者と管理者を兼ねることがあります
